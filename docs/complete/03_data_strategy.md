# 데이터 수집 전략: 성수동 오프셋 분석

## 타겟 지역
**성수동 / 연무장길** — 글로벌 핫스팟이자 외국인 관광객이 구글 지도만으로 길을 찾기 어려운 대표 지역

## 핵심 개념: 3-Source Triangulation

기존 방식(구글↔네이버 직접 비교)은 "어느 쪽이 맞는지" 알 수 없습니다.
우리는 **VWorld(국가 공식 좌표)**를 제3의 기준(Ground Truth)으로 도입하여
각 지도 소스의 오프셋을 독립적으로 측정합니다.

```
VWorld (정답) ← 기준점
  ├── Google 오프셋 = |Google좌표 - VWorld좌표|
  └── Naver 오프셋  = |Naver좌표  - VWorld좌표|

보정 벡터 = Google좌표 + (Naver오프셋 - Google오프셋)
→ 구글 지도에서 네이버 수준의 정밀도로 위치 안내
```

## 기준점(Anchor) 선정 원칙

| 원칙 | 이유 |
|------|------|
| 물리적 불변성 | 간판이나 상호는 바뀌지만, 역 출구와 교차로는 움직이지 않음 |
| 공간적 분포 | 성수동 전역에 고르게 분포하여 지역별 왜곡 패턴 포착 |
| 좌표 정밀도 | VWorld Geocoder가 도로명주소로 정확한 좌표 반환 |

## ML 학습 시 Feature Engineering 제안

학습 입력:
- `google_lat`, `google_lng` (구글 WGS84 좌표)
- `distance_to_nearest_anchor` (가장 가까운 기준점까지의 거리)
- `bearing_to_nearest_anchor` (기준점 방향각)

학습 타겟:
- `delta_lat`, `delta_lng` (구글↔네이버 좌표 차이)

모델:
- cuML `XGBRegressor` 또는 `RandomForestRegressor` (GPU 가속)
